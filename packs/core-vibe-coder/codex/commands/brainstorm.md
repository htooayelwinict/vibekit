# /brainstorm

Explore options and trade-offs before committing to an implementation path.

## Arguments

- `question`: design, architecture, naming, UX, or strategy decision to explore.

## Workflow

1. Frame the decision and constraints.
2. Generate a few meaningfully different options.
3. Compare benefits, costs, risks, reversibility, and verification burden.
4. Recommend a path when appropriate.
5. Write a durable summary in `brainstorm/` or the active plan if it should guide implementation.

## Escalation

Delegate to `brainstormer` for deeper trade-off exploration.
