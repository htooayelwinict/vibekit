# /docs

Update documentation, setup notes, or workflow guidance to match project behavior.

## Arguments

- `request`: documentation target, behavior change, setup flow, or missing explanation.

## Workflow

1. Inspect existing docs and nearby source behavior.
2. Update the smallest relevant documentation surface.
3. Keep commands, paths, and environment variables concrete.
4. Avoid documenting speculative behavior.
5. Verify links, commands, or generated examples when practical.

## Escalation

Delegate to `project-manager` when documentation must stay synchronized with broader planning work.
