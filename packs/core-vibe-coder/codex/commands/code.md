# /code

Implement planned work phase by phase.

## Arguments

- `request`: implementation task or active plan folder.

## Workflow

1. Read the active `plan.md` when one exists.
2. Identify the current phase and target files.
3. Make the smallest viable implementation changes.
4. Update phase notes or status when the plan is durable project context.
5. Run targeted verification and summarize results.

## Escalation

Delegate to `fullstack-developer` for multi-file implementation.
